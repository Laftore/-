    API оформления заказов
    Входными данными считаем массив пришедший методом POST из корзины в виде ассоциативного массива формата JSON ($sessionCartData).
    Класс Order работает с заказами:
        - метод getData() принимает данные и выделяет необходимые данные
        - метод record добавляет заказ в бд и выводит статус о выполненение/ошибке записи
        - метод edit Записывает статус выполнения заказа в бд
        - метод sendTo вызывается при успешном выполнении record() и отправляет письмо клиенту.

    При вызове любого метода класса Db сначала идет проверка подключения к бд методом checkConnection().
